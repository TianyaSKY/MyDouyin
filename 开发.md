# 短视频推荐系统开发文档（MVP）

> 技术栈：MySQL + Milvus + Redis + MinIO + Spring Boot + RabbitMQ + Torch(FastAPI) + React

本项目目标是实现一个**可跑通、可演进的短视频推荐系统最小闭环（MVP）**，覆盖：内容存储、召回、排序、用户行为采集、用户向量更新与前端展示。

---

## 1. 系统目标与范围

### 1.1 MVP 功能目标

* 首页 Feed 返回 20 条短视频
* 支持用户行为上报（曝光 / 完播 / 点赞 / 不感兴趣）
* 基于用户行为进行个性化推荐
* 冷启动用户可正常刷内容

### 1.2 非目标（MVP 阶段不做）

* 实时大规模训练
* 复杂审核系统
* 多端（仅 Web）

---

## 2. 总体架构

```
React Frontend
      |
Spring Boot (feed-service)
      |
-------------------------------------------------
|           |            |            |          |
Redis     MySQL       RabbitMQ     Milvus     MinIO
  |           |            |            |
  |        行为日志     事件流       向量召回     视频文件
  |
用户短期向量
```

### 服务划分

* **feed-service（Spring Boot）**：对外 API、候选合并、排序、重排、缓存
* **ml-service（FastAPI + Torch）**：排序模型 / 向量生成
* **event-consumer**：MQ 消费，更新用户特征与日志

---

## 3. 数据流设计

### 3.1 Feed 请求流程

1. 前端请求 `GET /feed?userId=xxx`
2. feed-service：

    * 从 Redis 获取用户短期兴趣向量
    * 热度召回（Redis/MySQL）
    * Milvus 向量召回（TopK）
    * 合并候选、去重
    * 调用 ml-service 排序（可选）
    * 重排打散
3. 返回 20 条视频
4. 异步发送曝光事件到 MQ

### 3.2 行为上报流程

1. 前端 `POST /event`
2. feed-service 写入 RabbitMQ
3. 消费者：

    * 更新 Redis 用户短期向量
    * 写 MySQL 行为日志

## 4. 数据存储设计

### 4.1 MySQL 表结构

#### user_profile

*   user_id BIGINT PK
*   username VARCHAR(64) UNIQUE
*   password VARCHAR(255) (BCrypt)
*   nickname VARCHAR(64)
*   avatar_url VARCHAR(512)
*   long_vec JSON (长期兴趣向量)
*   created_at DATETIME

#### video

* id BIGINT PK
* author_id BIGINT
* title VARCHAR(255)
* tags JSON
* cover_url VARCHAR(512)
* video_url VARCHAR(512)
* status TINYINT
* created_at DATETIME

#### user_event

* id BIGINT PK
* user_id BIGINT
* video_id BIGINT
* event_type ENUM('impression','finish','like','dislike')
* watch_ms INT
* ts DATETIME

#### video_stats_daily

* video_id BIGINT
* date DATE
* impr_cnt INT
* like_cnt INT
* finish_cnt INT
* watch_time_sum BIGINT

---

### 4.2 Redis Key 设计

* `video:hot` → ZSET（热度榜）
* `user:shortvec:{userId}` → HASH / JSON（短期兴趣向量）
* `user:seen:{userId}` → SET（最近已看视频）
* `feed:cache:{userId}` → LIST（短期 Feed 缓存）

---

### 4.3 Milvus 向量库

Collection：`video_embedding`

* video_id (Int64, PK)
* embedding (FloatVector, dim=128)
* tag_ids (Int64, optional)

索引：HNSW

* metric_type: COSINE
* topK: 200 ~ 1000

---

### 4.4 MinIO

* Bucket：`video-content`
* 存储：视频文件、封面图片
* video 表中仅保存访问 URL

## 5. 接口设计

### 5.0 认证接口 (已完成)

**POST /api/auth/register**

*   注册新用户
*   Req: `{"username": "...", "password": "...", "nickname": "..."}`
*   Res: `200 OK`

**POST /api/auth/login**

*   用户登录，获取 JWT Token
*   Req: `{"username": "...", "password": "..."}`
*   Res: `{"token": "Bearer eyJhbGciOi..."}`

**GET /api/auth/me**

*   获取当前登录用户信息
*   Header: `Authorization: Bearer <token>`
*   Res: `{"userId": 1, "username": "...", "nickname": "..."}`

## 8. 排序设计

### 8.1 规则排序（初期）

```
score = 热度分 + 兴趣匹配分 - 负反馈惩罚 + 新鲜度
```

### 8.2 模型排序（进阶）

* FastAPI `/rank`
* 输入：用户向量 + 候选视频特征
* 输出：score
* Torch 实现 MLP / LR
