# 短视频推荐系统开发文档（MVP）

> 技术栈：MySQL + Milvus + Redis + MinIO + Spring Boot + RabbitMQ + FastAPI(Python/Torch) + React

本文档用于记录 Douyin 项目当前 MVP 的架构、数据模型、接口设计与开发里程碑。

---

## 1. 系统目标与范围

### 1.1 MVP 目标

- 首页 Feed 返回 20 条短视频
- 支持用户行为上报（曝光、完播、点赞、分享）
- 基于用户行为做基础个性化推荐
- 支持视频上传（分片上传、断点续传、文件哈希秒传）

### 1.2 非目标（MVP 暂不做）

- 实时大规模在线训练
- 完整内容审核中台
- 多端（当前以 Web 为主）

---

## 2. 总体架构

```text
React Frontend
      |
Spring Boot (backend)
      |
-------------------------------------------------
|           |            |            |          |
Redis      MySQL       RabbitMQ     Milvus     MinIO
  |           |            |            |
  |        行为日志       事件流         向量召回     视频对象存储
  |
用户短期兴趣向量
```

### 服务划分

- `backend`（Spring Boot）：对外 API、鉴权、视频/用户元数据管理、上传流程编排
- `recommend`（FastAPI + Torch）：向量生成、排序服务
- `consumer`（后续）：消费 MQ，更新特征与落库

## 3. 数据流设计

### 3.1 Feed 请求流程

1. 前端请求 `GET /feed?userId=xxx`
2. backend 聚合候选：Redis 热度、MySQL、Milvus
3. 调用 recommend 服务进行排序（可选）
4. 返回 20 条视频
5. 异步发送曝光事件到 MQ

### 3.2 行为上报流程

1. 前端 `POST /event`
2. backend 写入 RabbitMQ
3. 消费者更新 Redis 特征并写 MySQL 行为日志

### 3.3 视频上传流程（已实现）

1. 前端本地计算文件哈希（支持 `MD5(32)` 或 `SHA-256(64)`）
2. 调 `POST /api/videos/upload/init`：
   - 命中哈希：直接返回 `instantUpload=true` 与 `videoUrl`（秒传）
   - 未命中：返回 `uploadId` 与已上传分片列表（断点续传）
3. 前端逐片上传 `POST /api/videos/upload/chunk`
4. 前端调 `POST /api/videos/upload/complete` 触发合并与哈希校验
5. 返回 `videoUrl` 后，再调用 `POST /api/videos` 写视频元数据

## 4. 数据存储设计

### 4.1 MySQL 关键表

- `user_profile`：用户信息
- `video`：视频元数据（标题、封面、地址、状态）
- `video_stats_daily`：视频日维度统计
- `user_event`：行为日志
- `file_asset`：文件哈希与视频 URL 映射（用于秒传）

### 4.2 Redis Key（建议）

- `video:hot`：ZSET，热视频分值
- `user:shortvec:{userId}`：用户短期兴趣向量
- `user:seen:{userId}`：用户最近已看视频集合
- `feed:cache:{userId}`：短期 Feed 缓存

### 4.3 Milvus

- Collection：`video_embedding`
- 主键：`video_id`
- 向量：`embedding (dim=128)`
- 索引：HNSW

### 4.4 对象存储

- 生产建议：MinIO
- 当前后端上传实现：本地磁盘 `storage/videos`（通过 `/uploads/videos/**` 暴露访问）

## 5. 接口设计

### 5.0 认证接口（已完成）

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/auth/me`

### 5.1 视频上传接口（已完成）

#### 1) 初始化上传

- `POST /api/videos/upload/init`
- 请求示例：

```json
{
  "fileName": "demo.mp4",
  "fileHash": "<md5-32或sha256-64>",
  "fileSize": 10485760,
  "chunkSize": 2097152,
  "totalChunks": 5
}
```

- 响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "instantUpload": false,
    "uploadId": "<fileHash>_10485760_5",
    "uploadedChunks": [0,1],
    "videoUrl": null
  }
}
```

#### 2) 上传分片

- `POST /api/videos/upload/chunk`
- `Content-Type: multipart/form-data`
- 字段：`uploadId`, `chunkIndex`, `chunk`

#### 3) 完成上传

- `POST /api/videos/upload/complete`
- 请求示例：

```json
{
  "uploadId": "<fileHash>_10485760_5",
  "fileName": "demo.mp4",
  "fileHash": "<md5-32或sha256-64>",
  "fileSize": 10485760,
  "totalChunks": 5
}
```

- 响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "fileHash": "<hash>",
    "videoUrl": "/uploads/videos/<hash>.mp4"
  }
}
```

### 5.2 视频元数据接口（已完成）

- `POST /api/videos`：创建视频元数据（新视频默认 `REVIEW`）
- `GET /api/videos`：分页查询
- `GET /api/videos/{id}`：按 ID 查询
- `PUT /api/videos/{id}`：更新元数据
- `PUT /api/videos/{id}/status`：更新状态
- `DELETE /api/videos/{id}`：删除

---

## 6. 安全与约束

- 上传接口需要登录态（JWT）
- 静态视频访问路径 `/uploads/videos/**` 放行
- 服务端会在 `complete` 阶段校验分片完整性、文件大小、文件哈希
- 严禁提交生产密钥，统一使用环境变量
