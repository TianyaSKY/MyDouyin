# 短视频推荐系统开发文档（MVP）

> 技术栈：MySQL + Milvus + Redis + MinIO + Spring Boot + RabbitMQ + FastAPI(Python/Torch) + React

本文档用于记录 Douyin 项目当前 MVP 的架构、数据模型、接口设计与开发里程碑。

---

## 1. 系统目标与范围

### 1.1 MVP 目标

- 首页 Feed 返回 20 条短视频
- 支持用户行为上报（曝光、完播、点赞、分享）
- 基于用户行为做基础个性化推荐
- 支持视频上传（分片上传、断点续传、文件哈希秒传）

### 1.2 非目标（MVP 暂不做）

- 实时大规模在线训练
- 完整内容审核中台
- 多端（当前以 Web 为主）

### 推荐系统架构（三阶段）

```
┌─────────────────────────────────────────────┐
│ 阶段1: 多路召回 (Spring Boot)               │
│ ├─ 热门池召回 (50%) - Redis ZSET           │
│ ├─ 标签召回 (30%) - 用户兴趣标签           │
│ └─ 向量召回 (20%) - Milvus ANN + 用户向量  │
│    (用户向量由 FastAPI 计算)                │
│                                              │
│ 召回 60 个候选视频                           │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│ 阶段2: 粗排 (Spring Boot)                   │
│ 规则排序：召回分数 + 热度分                  │
│ 热度分 = 点赞×2 + 完播×3 + 分享×5           │
│                                              │
│ 保留 Top 30-40                               │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│ 阶段3: 精排 (FastAPI - 可选)                │
│ 模型：Wide & Deep (PyTorch)                 │
│ 输出：点击率预估 (0-1)                       │
│                                              │
│ 返回 Top 20                                  │
└─────────────────────────────────────────────┘
```

### 服务划分

- `backend`（Spring Boot）：
  - 对外 API、鉴权、视频/用户元数据管理
  - 视频上传流程编排
  - 推荐召回（多路召回）与粗排
  - 行为上报与 MQ 生产
  
- `recommend`（FastAPI + PyTorch）：
  - 视频向量生成（VideoEncoder）
  - 用户向量计算（UserEncoder）
  - 精排服务（RankingModel - Wide & Deep）
  - 模型管理与推理
  
- `consumer`（Spring Boot 内置）：
  - 消费 MQ 行为事件
  - 更新用户画像（标签+向量）
  - 更新视频统计

## 3. 数据流设计

### 3.1 Feed 请求流程

1. 前端请求 `GET /api/feed?userId=xxx&size=20`
2. backend 多路召回：
   - 热门池召回（50%）：从 Redis ZSET 获取
   - 标签召回（30%）：基于用户兴趣标签匹配
   - 向量召回（20%）：用户融合向量 → Milvus ANN 检索
3. 去重过滤：过滤用户已看视频（Redis Set）
4. 粗排：召回分数 + 热度分
5. 返回 Top 20 视频
6. 异步发送曝光事件到 MQ

### 3.2 行为上报流程

1. 前端 `POST /api/event`（点击、点赞、完播、分享）
2. backend 写入 RabbitMQ
3. 消费者处理：
   - 保存行为日志到 MySQL
   - 更新视频统计（video_stats_daily）
   - 更新用户兴趣标签（interest_tags）
   - 调用 FastAPI 更新用户实时向量（Redis）

**用户向量更新流程**：
```
用户行为 → RabbitMQ → UserEventConsumer
                         ↓
                   UserEmbeddingService
                         ↓
              获取最近50个交互行为
                         ↓
              调用 FastAPI 批量获取视频向量
              POST /api/embedding/video/batch
                         ↓
              调用 FastAPI 计算用户向量
              POST /api/embedding/user
                         ↓
              存入 Redis (user:vec:{userId}, 24h TTL)
```

### 3.3 视频上传流程（已实现）

1. 前端本地计算文件哈希（支持 `MD5(32)` 或 `SHA-256(64)`）
2. 调 `POST /api/videos/upload/init`：
   - 命中哈希：直接返回 `instantUpload=true` 与 `videoUrl`（秒传）
   - 未命中：返回 `uploadId` 与已上传分片列表（断点续传）
3. 前端逐片上传 `POST /api/videos/upload/chunk`
4. 前端调 `POST /api/videos/upload/complete` 触发合并与哈希校验
5. 返回 `videoUrl` 后，再调用 `POST /api/videos` 写视频元数据

## 4. 数据存储设计

### 4.1 MySQL 关键表

- `user_profile`：用户信息
  - `interest_tags`：用户兴趣标签及权重（JSON）
  - `long_vec`：用户长期兴趣向量（JSON，128维）
- `video`：视频元数据（标题、封面、地址、状态）
  - `tags`：视频标签（JSON 数组）
- `video_stats_daily`：视频日维度统计
- `user_event`：行为日志
- `file_asset`：文件哈希与视频 URL 映射（用于秒传）

### 4.2 Redis Key

- `video:hot`：ZSET，热视频分值（Top 1000）
- `user:vec:{userId}`：用户短期兴趣向量（List，128维，24小时过期）
- `user:seen:{userId}`：用户最近已看视频集合（Set，7天过期）
- `feed:cache:{userId}`：短期 Feed 缓存（可选）

### 4.3 Milvus

- Collection：`video_embedding`
- 主键：`video_id`
- 向量：`embedding (dim=128)`
- 索引：HNSW

### 4.4 对象存储

- 生产建议：MinIO
- 当前后端上传实现：本地磁盘 `storage/videos`（通过 `/uploads/videos/**` 暴露访问）

### 5.1 视频上传接口（已完成）

- `POST /api/videos/upload/init`

```json
{
  "fileName": "demo.mp4",
  "fileHash": "<md5-32或sha256-64>",
  "fileSize": 10485760,
  "chunkSize": 2097152,
  "totalChunks": 5
}
```

- 响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "instantUpload": false,
    "uploadId": "<fileHash>_10485760_5",
    "uploadedChunks": [0,1],
    "videoUrl": null
  }
}
```

#### 2) 上传分片

- `POST /api/videos/upload/chunk`
- `Content-Type: multipart/form-data`
- 字段：`uploadId`, `chunkIndex`, `chunk`

#### 3) 完成上传

- `POST /api/videos/upload/complete`
- 请求示例：

```json
{
  "uploadId": "<fileHash>_10485760_5",
  "fileName": "demo.mp4",
  "fileHash": "<md5-32或sha256-64>",
  "fileSize": 10485760,
  "totalChunks": 5
}
```

- 响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "fileHash": "<hash>",
    "videoUrl": "/uploads/videos/<hash>.mp4"
  }
}
```

### 5.2 视频元数据接口（已完成）

- `POST /api/videos`：创建视频元数据（新视频默认 `REVIEW`）
- `GET /api/videos`：分页查询
- `GET /api/videos/{id}`：按 ID 查询
- `PUT /api/videos/{id}`：更新元数据
- `PUT /api/videos/{id}/status`：更新状态
- `DELETE /api/videos/{id}`：删除

### 5.3 Feed 推荐接口（已完成）

- `GET /api/feed?userId={userId}&size={size}`：获取个性化推荐视频

**多路召回策略**：
- 热门池召回（50%）：从 Redis ZSET 获取高热度视频
- 标签召回（30%）：基于用户兴趣标签匹配视频
- 向量召回（20%）：通过用户向量在 Milvus 中检索相似视频

**排序算法**：
```
最终分数 = 召回分数 + 热度分
热度分 = 点赞数×2 + 完播数×3 + 分享数×5
```

### 5.4 用户标签接口（已完成）

- `GET /api/user-tags/{userId}?topN=10`：获取用户兴趣标签
- `POST /api/user-tags/{userId}/decay?decayFactor=0.95`：手动触发标签衰减

**标签更新机制**：
- 用户每次点击、点赞、完播、分享视频时自动更新
- 不同行为权重：分享(2.0) > 完播(1.5) > 点赞(1.0) > 点击(0.3)
- 每天凌晨2点自动衰减所有用户标签（衰减2%）

### 5.5 用户向量接口（已完成）

- `GET /api/user-embedding/{userId}/realtime`：获取用户实时向量（短期兴趣）
- `GET /api/user-embedding/{userId}/longterm`：获取用户长期向量
- `GET /api/user-embedding/{userId}/fused?shortTermWeight=0.7`：获取融合向量
- `POST /api/user-embedding/{userId}/calculate`：手动触发向量计算
- `PUT /api/user-embedding/{userId}/longterm`：更新长期向量（供离线任务调用）

**向量更新机制**：
- 实时向量：基于最近50个交互行为加权平均计算，存储在 Redis（24小时过期）
- 长期向量：通过离线训练定期更新，存储在 MySQL
- 融合策略：`融合向量 = 短期向量×0.7 + 长期向量×0.3`
- 时间衰减：`e^(-0.01×小时数)`，越近的行为权重越高

## 8. 用户画像与推荐特征

### 8.1 用户兴趣标签

**存储结构**：
```json
{
  "科技": 0.35,
  "美食": 0.28,
  "旅游": 0.15,
  "音乐": 0.12,
  "运动": 0.10
}
```

**更新流程**：
```
用户行为 → RabbitMQ → UserEventConsumer → UserTagService
                                              ↓
                                    提取视频标签 + 计算权重增益
                                              ↓
                                    归一化 + 裁剪（最多50个标签）
                                              ↓
                                    更新到 MySQL user_profile.interest_tags
```

**行为权重**：
- 曝光：+0.1
- 点击：+0.3
- 点赞：+1.0
- 完播：+1.5
- 分享：+2.0

**衰减机制**：
- 定时任务：每天凌晨2点执行
- 衰减因子：0.98（每天衰减2%）
- 最小阈值：权重低于0.01的标签自动移除

### 8.2 用户兴趣向量（Embedding）

**双向量架构**：

1. **短期向量（实时）**
   - 存储：Redis `user:vec:{userId}`
   - 维度：128
   - 更新：每次用户行为后自动更新
   - 过期：24小时
   - 用途：捕捉短期兴趣变化

2. **长期向量（离线）**
   - 存储：MySQL `user_profile.long_vec`
   - 维度：128
   - 更新：定期通过离线训练
   - 用途：稳定的长期兴趣

**实时向量计算公式**：
```
用户向量 = Σ(视频向量 × 行为权重 × 时间衰减) / Σ权重

时间衰减 = e^(-0.01 × 小时数)
```

**向量融合**：
```
融合向量 = 短期向量 × 0.7 + 长期向量 × 0.3
```

**更新流程**：
```
用户行为 → RabbitMQ → UserEventConsumer → UserEmbeddingService
                                              ↓
                                    获取最近50个交互行为
                                              ↓
                                    调用 FastAPI 批量获取视频向量
                                    POST /api/embedding/video/batch
                                              ↓
                                    调用 FastAPI 计算用户向量
                                    POST /api/embedding/user
                                              ↓
                                    存入 Redis（24小时过期）
```

### 8.3 Feed 推荐流程

```
1. 多路召回
   ├─ 热门池召回（50%）：Redis ZSET Top N
   ├─ 标签召回（30%）：基于用户兴趣标签匹配视频
   └─ 向量召回（20%）：用户融合向量 → Milvus ANN 检索

2. 去重过滤
   └─ 过滤用户已看视频（Redis Set）

3. 粗排
   └─ 最终分数 = 召回分数 + 热度分

4. 截断返回
   └─ 返回 Top 20

5. 异步处理
   ├─ 发送曝光事件到 MQ
   └─ 更新用户已看集合
```

### 11.3 模型定义

**VideoEncoder** - 视频编码器
```python
输入: 视频标签ID
输出: 128维向量
用途: 视频特征提取
```

**UserEncoder** - 用户编码器
```python
输入: 视频向量 + 行为权重
输出: 128维用户向量
用途: 用户兴趣建模
```

**RankingModel** - Wide & Deep 精排模型
```python
输入: 用户向量 + 视频向量 + 召回分数 + 热度分
输出: 点击率预估 (0-1)
用途: 精排优化
```
